# Glossaire debutant

Ce glossaire explique les mots que je peux utiliser dans ce projet.

Le but est simple :

- t'aider a suivre sans jargon inutile
- te donner un vocabulaire de base
- rendre les echanges plus confortables

## Les mots les plus importants

### Application cible

L'application cible, c'est le site ou l'application web que l'on observe.

Ici, l'application cible est `FSVB Studio Portfolio`.

### Agent

Un agent est un profil de travail specialise.

Il aide a regarder le projet sous un angle precis, par exemple :

- experience utilisateur
- qualite
- performance
- synthese

### Analyse

Une analyse, c'est le fait d'observer le projet pour comprendre son fonctionnement, ses points forts et ses problemes possibles.

### Accessibilite

L'accessibilite, c'est le fait de rendre le site utilisable par le plus grand nombre, y compris des personnes avec des besoins visuels, moteurs ou autres.

Exemple :
bonne lisibilite, navigation clavier, textes clairs pour les formulaires.

### Audit

Un audit est une analyse un peu plus structuree, souvent centree sur un sujet precis :

- audit UX
- audit design
- audit performance
- audit qualite
- audit accessibilite
- audit SEO
- audit securite

### Bug

Un bug est un probleme dans le fonctionnement normal de l'application.

Exemple :
un bouton ne repond pas ou une page affiche une erreur.

### Capture

Une capture est une image ou un element visuel garde comme preuve.

### Code

Le code, c'est l'ensemble des fichiers qui disent a l'application comment fonctionner.

### Composant

Un composant est un morceau d'interface reutilisable.

Exemple :
un bouton, un menu, une carte, un formulaire.

### Confirme

Quand je dis "confirme", cela veut dire que le point a ete verifie.

### Depot

Le depot, c'est le dossier principal du projet.

Il contient :

- les fichiers du projet
- la documentation
- les outils
- les rapports

On dit aussi parfois "repository" ou "repo".

### Documentation

La documentation, ce sont les textes qui expliquent le projet, son fonctionnement et la facon de l'utiliser.

### Dossier

Un dossier est un emplacement qui sert a ranger plusieurs fichiers.

### Environnement local

L'environnement local, c'est ton ordinateur configure pour faire tourner le projet.

Cela comprend par exemple :

- les logiciels installes
- les dependances
- les variables de configuration

### Evidence

Le mot "evidence" dans ce projet veut dire "preuve".

Cela peut etre :

- une capture d'ecran
- un journal technique
- une sortie de commande
- une note d'observation

### Fichier

Un fichier est un document unique dans le projet.

Exemple :
un fichier texte, un script, une image, une page de documentation.

### Frontend

Le frontend, c'est la partie visible de l'application, celle que l'utilisateur voit et utilise dans son navigateur.

### Hypothese

Quand je dis "hypothese", cela veut dire que c'est une piste probable, mais pas encore totalement verifiee.

### Interface

L'interface, c'est ce que l'utilisateur voit a l'ecran :

- textes
- boutons
- images
- formulaires
- menus

### Indexabilite

L'indexabilite, c'est la capacite d'une page a etre lue et prise en compte par un moteur de recherche.

### Journal

Un journal est une trace ecrite de ce qu'il s'est passe.

On dit aussi parfois "log".

### Jargon

Le jargon, ce sont des mots techniques pas toujours utiles pour comprendre l'essentiel.

Dans ce depot, on essaie justement d'en utiliser le moins possible.

### Livrable

Un livrable est un document ou un resultat prepare pour etre partage.

Exemple :
une synthese, un rapport, une note de conclusion.

### Local

Quand je dis "local", cela veut dire "sur ton ordinateur", et non sur un serveur distant.

### Metadonnees

Les metadonnees sont de petites informations attachees a une page web.

Elles aident par exemple les moteurs de recherche ou les apercus de partage a mieux comprendre la page.

### MCP

MCP est un moyen pour relier l'assistant a certains outils.

Dans ce projet, cela peut servir a :

- lancer ou verifier l'application
- ouvrir un navigateur outille
- collecter certaines informations utiles

Tu peux simplement retenir :
"MCP = passerelle vers des outils utiles".

### Mission

Une mission, c'est un travail a faire sur un sujet donne.

Exemple :
tester la page d'accueil ou verifier un parcours utilisateur.

### Observation

Une observation est un constat releve pendant l'analyse.

Exemple :
"le texte du bouton n'est pas tres clair".

### Navigation clavier

La navigation clavier, c'est le fait de pouvoir utiliser le site sans souris, seulement avec le clavier.

### Parcours utilisateur

Un parcours utilisateur, c'est une suite d'etapes que suit une personne dans l'application.

Exemple :
arriver sur la page d'accueil, cliquer sur une rubrique, puis envoyer un formulaire.

### Performance

La performance, c'est la rapidite et la fluidite percue de l'application.

### Preuve

Une preuve est un element garde pour montrer qu'un constat repose sur quelque chose de reel.

Exemple :
une capture, un journal, une note horodatee.

### Probable

Quand je dis "probable", cela veut dire que c'est credible, mais pas encore confirme avec certitude.

### Projet

Le projet, c'est l'ensemble du travail organise dans ce depot.

### Qualite

La qualite, ici, c'est l'idee generale de fiabilite et de coherence.

On cherche par exemple :

- des erreurs visibles
- des comportements inattendus
- des incoherences entre pages

### Rapport

Un rapport est un document qui rassemble les constats, les preuves et les recommandations.

### Recommandation

Une recommandation est une action conseillee apres l'analyse.

Exemple :
clarifier un message, corriger un bouton, tester un autre parcours.

### Reference

Dans `docs/reference/`, une reference est un document descriptif qui explique le fonctionnement du projet.

### Restitution

La restitution, c'est la facon de presenter les resultats.

Exemple :
une synthese courte, un rapport detaille, une liste priorisee.

### Risque

Un risque est un probleme possible pour l'utilisateur, le projet ou la comprehension.

Exemple :
un ecran confus peut faire perdre confiance.

### SEO

SEO designe le referencement naturel.

En simple :
c'est tout ce qui aide un site a etre mieux compris et trouve sur les moteurs de recherche.

### Securite

La securite concerne la protection du site, des donnees et des usages contre les risques ou les mauvaises expositions.

### Scenario

Un scenario est un cas d'usage concret que l'on teste.

Exemple :
"un visiteur consulte le portfolio puis cherche a contacter le studio".

### Script

Un script est un petit fichier qui automatise une tache.

Exemple :
lancer l'application ou collecter des preuves.

### Stack

Le mot "stack" designe les technologies principales utilisees par le projet.

Ici, cela peut inclure par exemple React, TypeScript et Vite.

Tu peux retenir :
"stack = boite a outils technique principale du projet".

### Synthese

Une synthese est une version courte et claire des points les plus importants.

### Template

Un template est un modele pret a remplir.

Exemple :
un modele de rapport.

### Test

Un test, c'est une verification.

Dans ce projet, cela peut etre :

- essayer un parcours
- verifier qu'une page se charge
- confirmer qu'un comportement est correct

### Outil

Un outil est un moyen pratique pour faire une action.

Exemple :
un script, un navigateur outille, une commande de verification.

### UX

UX veut dire "experience utilisateur".

Cela concerne la facon dont une personne comprend et vit l'utilisation de l'application.

En simple :
est-ce que c'est clair, logique et confortable ?

### Variable d'environnement

Une variable d'environnement est une information de configuration stockee a part.

Exemple :
une cle technique ou une adresse de service.

Le fichier `.env` sert souvent a stocker ce type d'information.

### Verifier

Verifier, c'est controler si un point est vrai ou non.

### Workflow

Un workflow, c'est une facon de travailler pas a pas.

Exemple :

1. lancer l'application
2. observer une page
3. garder une preuve
4. rediger une synthese

## Mots que je peux aussi employer parfois

### React

React est un outil de construction d'interface web.

Tu peux le voir comme une facon d'assembler les ecrans en petits morceaux.

### TypeScript

TypeScript est un langage utilise pour structurer le code plus clairement et limiter certaines erreurs.

### Vite

Vite est un outil qui aide a lancer et construire l'application web rapidement pendant le travail local.

### Supabase

Supabase est un service externe souvent utilise pour les donnees, l'authentification ou d'autres fonctions utiles.

### PayPal

PayPal est un service de paiement en ligne.

## Regle simple pour lire mes messages

Quand je parle d'un point, j'essaierai de le ranger dans une de ces categories :

- confirme : verifie
- probable : plausible mais pas encore confirme
- a verifier : demande encore un controle

Si un mot reste flou, on peut l'ajouter ici au fur et a mesure.
