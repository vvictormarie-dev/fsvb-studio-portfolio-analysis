# Vue d'ensemble du depot

## A quoi sert ce projet

Ce depot sert a analyser l'application web `FSVB Studio Portfolio` sans la modifier par defaut.

L'objectif n'est pas d'abord de developper une nouvelle fonctionnalite.
L'objectif est surtout de :

- lancer l'application localement
- observer son fonctionnement
- tester des parcours
- noter les constats utiles
- garder des preuves
- produire des syntheses claires

## Idee generale

On peut voir ce depot comme un espace de travail organise pour repondre a trois besoins :

1. comprendre l'application existante
2. verifier ce qui fonctionne bien ou moins bien
3. transformer ces observations en documents faciles a partager

## Les dossiers principaux

### `fsvb-studio-portfolio/`

Contient la copie locale de l'application cible.

### `fsvb-studio-portfolio/portfolio-app/`

Contient l'application web principale detectee dans la copie locale.

### `docs/`

Contient les guides et documents de fonctionnement.

On y trouve notamment :

- `how-to/` : les pas a pas pratiques
- `templates/` : les modeles de rapport
- `workflows/` : la facon recommande de travailler
- `reference/` : les explications descriptives du projet

### `reports/`

Contient les resultats de l'analyse.

- `drafts/` : notes de travail et premieres observations
- `evidence/` : preuves, journaux et captures
- `generated/` : sorties produites par des outils
- `consolidated/` : documents finaux relus et partageables

### `tools/`

Contient de petits scripts utiles pour aider au quotidien.

## Facon simple de comprendre le fonctionnement

Le fonctionnement du depot suit souvent cette logique :

1. on prepare l'environnement
2. on lance ou observe l'application
3. on teste une page ou un parcours
4. on garde les preuves utiles
5. on redige une synthese lisible

## Ce qui est confirme

- ce depot est pense d'abord pour l'observation et la documentation
- l'application cible est analysee en priorite en lecture
- les rapports doivent rester lisibles pour un public non expert

## Ce qui est important a retenir

Si tu debutes, le plus utile est de voir ce depot comme un classeur bien range :

- l'application a observer
- les guides pour savoir quoi faire
- les espaces pour noter ce qu'on voit
- les profils d'analyse pour ne pas partir dans tous les sens
