# Guide simple des agents

## A quoi sert un agent ici

Dans ce depot, un agent est un profil de travail.

Il ne remplace pas le projet.
Il aide surtout a regarder l'application avec un angle precis.

Autrement dit, chaque agent pose une question differente :

- est-ce clair pour l'utilisateur ?
- est-ce beau et coherent ?
- est-ce qu'un parcours fonctionne de bout en bout ?
- y a-t-il des erreurs visibles ?
- est-ce que c'est lent ?
- est-ce que le site est accessible ?
- est-ce que le site est bien prepare pour le referencement ?
- y a-t-il des risques de securite visibles ?
- comment transformer tout cela en synthese claire ?

## Les agents disponibles

### `coordinator`

Role :
organiser la mission et choisir la bonne approche.

Quand l'utiliser :

- au debut d'une analyse
- quand il faut prioriser
- quand plusieurs observations doivent etre regroupees

En langage simple :
c'est le profil qui aide a mettre de l'ordre.

### `ux-audit`

Role :
observer si l'application est facile a comprendre et a utiliser.

Il regarde par exemple :

- la clarte des messages
- la comprehension des actions possibles
- les moments de doute ou d'hesitation
- les efforts demandes a l'utilisateur

En langage simple :
c'est le profil "est-ce simple pour la personne qui utilise le site ?"

### `design-review`

Role :
observer la qualite visuelle et la coherence des ecrans.

Il regarde par exemple :

- la lisibilite
- la hierarchie visuelle
- la coherence graphique
- l'impression generale de qualite

En langage simple :
c'est le profil "est-ce clair, propre et coherent visuellement ?"

### `user-journey-tester`

Role :
tester un scenario utilisateur du debut a la fin.

Il regarde par exemple :

- les prerequis
- les etapes importantes
- les points de blocage
- le resultat final obtenu ou non

En langage simple :
c'est le profil "est-ce que le parcours marche vraiment du debut a la fin ?"

### `quality-checker`

Role :
reperer les anomalies visibles et les incoherences.

Il regarde par exemple :

- les erreurs interface
- les messages incomplets
- les comportements inattendus
- les differences etranges entre pages

En langage simple :
c'est le profil "qu'est-ce qui semble cassé, fragile ou incoherent ?"

### `performance-investigator`

Role :
observer les lenteurs et les chargements lourds.

Il regarde par exemple :

- le temps de chargement percu
- les transitions lentes
- les pages ou moments plus lourds
- les parcours sensibles

En langage simple :
c'est le profil "ou est-ce que l'utilisateur peut ressentir de la lenteur ?"

### `accessibility-auditor`

Role :
verifier si l'application reste utilisable pour des personnes avec des besoins d'accessibilite varies.

Il regarde par exemple :

- la lisibilite et les contrastes
- la navigation au clavier
- la structure des pages
- les formulaires et leurs libelles

En langage simple :
c'est le profil "est-ce que le site reste utilisable par le plus grand nombre ?"

### `seo-reviewer`

Role :
observer si le site a de bonnes bases pour etre compris et trouve par les moteurs de recherche.

Il regarde par exemple :

- les titres de page
- les metadonnees importantes
- la structure du contenu
- la clarte des liens et du contenu visible

En langage simple :
c'est le profil "est-ce que le site est bien prepare pour etre visible sur le web ?"

### `security-reviewer`

Role :
reperer les risques visibles ou probables lies a la securite.

Il regarde par exemple :

- les informations sensibles exposees
- les comportements qui semblent risqués
- certaines configurations visibles cote navigateur
- les zones qui meritent une verification plus poussee

En langage simple :
c'est le profil "y a-t-il un risque visible pour la protection du site ou des donnees ?"

### `documentation-synthesizer`

Role :
transformer les observations en document clair.

Il aide a produire :

- une synthese executive
- les messages clefs
- les risques
- les actions recommandees

En langage simple :
c'est le profil qui transforme l'analyse en document partageable.

## Comment choisir simplement

Si tu ne sais pas lequel utiliser :

- pour organiser le travail : `coordinator`
- pour la facilite d'usage : `ux-audit`
- pour l'apparence visuelle : `design-review`
- pour tester un scenario complet : `user-journey-tester`
- pour chercher les problemes visibles : `quality-checker`
- pour les lenteurs : `performance-investigator`
- pour l'accessibilite : `accessibility-auditor`
- pour le referencement : `seo-reviewer`
- pour la securite : `security-reviewer`
- pour rediger une synthese : `documentation-synthesizer`

## Ce qu'il faut retenir

Les agents ne compliquent pas le projet.
Au contraire, ils servent a le rendre plus simple a parcourir.

Ils evitent de tout melanger en separant les questions :

- comprendre
- observer
- verifier
- resumer
